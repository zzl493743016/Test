刚才宝玉老师在哥飞的群里发了一张截图，有个叫 Mike Carson 的老外，2023年的3月花6万美金买下来了 Cursor.com 这个域名，一年后的5月份，以39.9万美元的价格卖给了AI编程工具 Cursor 。


哥飞就很好奇，这是一次撞大运的投资呢？还是一次眼光卓绝的投资呢？

于是，哥飞去梳理了一下 Cursor 早期的故事，下面以时间线的形式给大家讲一下。

Cursor 早期使用过两个域名，Cursor.so 和 Cursor.sh 。

Cursor.so 注册于2021年12月11日，Cursor.sh 注册于2023年4月5日，现在这两个域名，都跳转到了 Cursor.com。

在 Archive.org 查 Cursor.so ，可以看到最早的使用记录出现在2022年。

整个2022年，Archive.org 只记录了Cursor.so的2条信息，一条是1月20日，一条是11月12日。

2022年的1月20日，Cursor.so 长下面这样，从文案可以看到，这是一个用AI帮你智能写邮件的工具。

2022年的11月12日，Cursor.so 长下面这样，从文案可以看到，这是一个用AI帮你辅助制作CAD的工具。解释一下，CAD常用于结构设计，3D建模。

2022年的11月30日，ChatGPT 发布，此时可以调用 text-davinci-003 模型，2023年的3月1日，GPT3.5 API 发布。

2023年，Archive.org 记录了Cursor.so的信息就多了。

2023年1月18日，Cursor.so 发布了新的页面，这时候就已经转型定位成了AI辅助编程工具了。不过产品还没发布，需要大家填写邮箱，参与排队。

这时候的 Slogan 是 “Code editing, redefined：Refactor, understand, and write code effortlessly using Cursor's AI source editor”，用中文一句话概括就是“Cursor 定位于 AI 驱动的代码编辑器，让用户可以轻松地重构、理解和编写代码”。

哥飞也找到了Cursor创始人发的产品介绍推文，这条推文发布时间是2023年的1月18日。

再到2023年的3月16日，Cursor.so 上已经可以下载 Mac 版本的软件了。

此时的 Slogan 变成了“Build Software. Fast. Write, edit, and chat about your code with GPT-4”，强调了是基于GPT4驱动的AI工具，帮助用户快速的构建软件，可以写代码、修改代码以及与代码对话。

哥飞同样找到了创始人在3月16日发的介绍推文。

再看下谷歌趋势里“Cursor AI”这个词的搜索热度变化，可以看到在整个3月份，最早有变化是在3月17日，刚好就是在发布客户端之后。


刚才，哥飞去微信里搜索了“curosr.so”，发现最早的记录也是在3月17日。


老外 Mike Carson 只说了是2023年3月买的域名，没说具体是哪一天。

但是哥飞根据 Cursor.com 的 Whois 信息变动历史记录，大概确定了，应该是在3月14到3月18日左右。



这个时间点，刚好就是 Cursor.so 发布产品的时间。

而在 Archive.org 里查询 Cursor.com 的网页快照，可以看到在2月底和3月底都有记录。不过2月底的2条打开都是404页面。


而打开2023年3月22日的快照，就已经是一个域名停放加出售页面了。

直到2024年的5月，Mike Carson 才与 Cursor 团队达成了购买协议，2024年的12月，Cursor 才支付了全部款项。

从域名 Whois 变动可以看到，大概是在2024年5月31日交接的域名。所以5月份肯定是先付了一笔首付的，不然不可能交接域名。


2024年6月6日，Cursor.com 正式被 Cursor 启用。

换上新域名后，Cursor 就一飞冲天了。



当然，上面是开玩笑的说法，行内人都知道，真正的功劳是2024年6月21日发布的 Claude 3.5 Sonnet 。

所以，现在可以回答开头的提问了，“这是一次撞大运的投资呢？还是一次眼光卓绝的投资呢？”

哥飞倾向于，这是一位有着丰富域名投资经验的域名玩家，通过自己收集的信息，快速判断，这个域名有了终端之后，绝对会升值，于是赶紧花6万美金把域名给买下来了。

解释一下，这里的“终端”，指的是终端买家的意思。如果有哪家公司名称、产品名称、商标刚好跟域名一样，就会被认为是域名的潜在终端买家。

某种程度上来说，他比 Cursor 的创始人还更敢投资。毕竟2023年3月创始人都没想着要把 Cursor.com 给买回来呢。

创始人选择在2023年的4月5日，又注册了 Cursor.sh ，他可能觉得，程序员对于 sh 更有亲切感吧。

关于 Cursor 这个词，哥飞之前在社群里有另一个分享，哥飞也一起发出来给大家看下。


哥飞在搜索 Cursor 时发现一个有趣的事情，Cursor 这个AI代码编辑器的官网拿到了第一名。

而 Cursor 这个词本来表示的是光标，写前端的同学可能会经常用到这个单词。

在 Cursor.com 上线之前，肯定不是他拿到第一名的，但是他用了这么一个通用的名词当作自己的品牌词，最终也成功抢下来了这个词。

谷歌给 Cursor.com 出了 Sitelinks ，就说明谷歌认可 Cursor 这个词是 Cursor.com 的品牌词。

这给我们取品牌词也有一些启发，其实也可以用一些单词来做品牌词，只要暂时这个单词还没有被别的大站使用，就可以了。

类似的，Notion 也是一个通用单词，但是最后也变成了品牌词。


这个自定义网页里鼠标光标的浏览器插件有400万的周活。这是大家没想到的需求吧。

https://chromewebstore.google.com/detail/custom-cursor-for-chrome/ogdlpmhglpejoiomcodnpjnfgcpmgale?hl=en-US


差不多5万个评价，还能保持在4.7分，说明大家都很满意。

官网 custom-cursor.com 一年有2028万访问量。


官网做得挺好的，为每一个光标都做了一个内页。并且当你没有安装浏览器插件时，会提醒你安装。

也就是说，每一个内页，都能够从谷歌搜索获取流量，最后都被引导到安装浏览器插件。

这真是特别流畅，特别妙的留存用户的方式。

大家发现没有，虽然这个插件本身都是免费使用的，但是配套的官网放了谷歌广告。

在官网里引导用户安装浏览器插件，在插件里引导用户打开官网查看更多的光标。

从搜索引擎获取全新的访客，沉淀到插件。

插件又给官网引流，之后在官网靠广告变现。

分析到这里，是不是盈利模式就很明确了。

这个方式真的很自然，用户的所有操作都发生在浏览器里，在插件和网站之间互相跳转几乎没有任何成本。

我们做网站，最担心的就是用户留存问题，而浏览器插件就帮助你解决了留存问题。

可以思考一下，还有哪些需求可以用这个模式来做。
